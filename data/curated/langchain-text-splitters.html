What are Text Splitters?

Text splitters break large documents into smaller chunks suitable for embedding and retrieval. Proper chunking is crucial for RAG performance - chunks must be small enough to be precise but large enough to contain meaningful context.

Key Concepts

  • Chunk Size: Target length of each chunk (in characters or tokens)
  • Chunk Overlap: How much consecutive chunks share (preserves context)
  • Separators: Characters/patterns used to split (paragraphs, sentences, etc.)

Common Splitter Types

RecursiveCharacterTextSplitter

The recommended default. Tries to split on paragraphs, then sentences, then words.

import { RecursiveCharacterTextSplitter } from "langchain/text_splitter";

const splitter = new RecursiveCharacterTextSplitter({
    chunkSize: 1000,
    chunkOverlap: 200,
    separators: ["\n\n", "\n", " ", ""]
});

const chunks = await splitter.splitText(document);

CharacterTextSplitter

Simple splitting on a single separator.

import { CharacterTextSplitter } from "langchain/text_splitter";

const splitter = new CharacterTextSplitter({
    separator: "\n\n",
    chunkSize: 1000,
    chunkOverlap: 200
});

TokenTextSplitter

Splits based on token count (important for LLM context limits).

import { TokenTextSplitter } from "langchain/text_splitter";

const splitter = new TokenTextSplitter({
    chunkSize: 500,      // tokens
    chunkOverlap: 50     // tokens
});

MarkdownTextSplitter

Respects Markdown structure (headers, code blocks, lists).

import { MarkdownTextSplitter } from "langchain/text_splitter";

const splitter = new MarkdownTextSplitter({
    chunkSize: 1000,
    chunkOverlap: 200
});

Choosing Chunk Size

  • Small chunks (200-500): More precise retrieval, may lose context
  • Medium chunks (500-1000): Good balance for most use cases
  • Large chunks (1000-2000): More context, may retrieve irrelevant content

Best Practices

  • Use 10-20% overlap to preserve context across chunks
  • Match chunk size to embedding model's optimal input
  • Use semantic splitters for better chunk boundaries
  • Test different configurations with your actual queries
